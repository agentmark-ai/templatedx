## Accessing Variables

#### Dot Notation

Use dot notation to access object properties within the template context, similar to JavaScript.

```jsx
{ props.username }
```

---

#### Bracket Syntax

Bracket syntax allows access to object properties using string keys, which is useful for dynamic or complex property names.

```jsx
{ props['user-name'] }
```

## Error Handling

Accessing an undefined or null variable will throw an error, ensuring that all variables used are explicitly defined.

For example:

```jsx
{ props.undefinedVar }
```

This will result in an error: "Undefined variable 'undefinedVar' in template."

## Examples

#### Accessing a Defined Variable

```jsx
{ props.username }
```

`Alice`

---

#### Accessing Nested Properties

```jsx
{ props.user.firstName } { props.user.lastName }
```

`Alice Johnson`

---


#### Using Bracket Syntax for Dynamic Properties

```jsx
{ props['user-email'] }
```

alice.johnson@example.com

---

#### Error Scenario: Undefined Variable

```jsx
{ props.user.address.street }
```

**Error:** Cannot read property 'street' of undefined in template.
